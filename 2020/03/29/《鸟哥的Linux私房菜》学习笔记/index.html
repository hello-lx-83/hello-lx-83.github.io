《鸟哥的Linux私房菜》学习笔记

发表于 2020-03-29 更新于 2020-04-02

第一部分 Linux的规划与安装

第零章 计算机概论

  • 如果想要Linux帮我做好我想要达成的工作，则Linux的学习并不是只要“会用就好”
  1. 计算机：接受使用者输入指令与数据，经由中央处理器的数字与逻辑单元运算处理后，以产生或存储成有用的信息。
  2. 计算机硬件的五大单元：包括输入单元，cpu内部的控制单元，算数逻辑单元，内存，输出单元。其中，cpu和内存的关系划重点。
    1. cpu的架构
      • 精简指令集（RISC）
      • 复杂指令及（CISC）
    2. 运行流程
      • cpu（大脑）,内存（此刻），硬盘（回忆），主板（神经系统），周边设备（与外界互动），电源供应器（心脏）
    3. 常用计算单位
      • 1 Byte = 8bits
    4. cpu的工作频率（cpu每秒可以进行的工作次数）：外频与倍频（与缓冲区的概念相关）
    5. 32位与64位的cpu与总线宽度
    6. 内存
      • 动态随机存取内存（DRAM）：只有通电时才能记录与使用，断电后数据就消失了; DDR（双倍数据传送速度）：可以在一次工作周期中完成两次数据传送。
      • 多通道设计：传统总线宽度一般是64位，将两条64位的内存汇整在一起，就可以达到128位，这是双通道的设计。
      • 静态随机存取内存（SRAM）：高性能，速度快，一般用作高速缓存
      • 只读存储器（ROM）非挥发性内存，没有通电也能将数据记录下来
      • BIOS
    7. 显卡（VGA）：和图像影像显示相关
      • GPU：图像处理器，在显卡上嵌入一个加速芯片
      • HDMI 可以同时传送影像与声音
    8. 硬盘与存储设备
      • SSD（固态硬盘）
    9. 主板
      • CMOS：主要功能，记录主板上面的重要参数
      • BIOS：写入主板上面的程序，可以在开机的时候执行，以载入CMOS中的参数，并尝试调用存储设备中的开机程序，进一步进入操作系统中。BIOS也可以修改CMOS中的数据，每种主板调用BIOS设置程序的按键不同，详情百度。
  3. 软件程序运行:没有软件的计算机只是一堆废铁
    1. 操作系统:其实也是一组程序，这组程序的重点在于管理电脑的所有活动以及驱动系统中的所有硬件。
    2. 系统调用：为了让程序员比较容易开发软件，操作系统除了核心程序外，通常还会提供一整组开发接口，即系统调用层
    3. 核心功能：系统调用接口，程序管理，内存管理，文件系统管理，设备的驱动

第一章 Linux是什么/如何学习

好的书本带你上天堂，坏的书本让你穷瞎忙

  1. 主要介绍了Linux的发展历程
  2. 如何学习Linux
    • 从头学习Linux基础，基础知识是学习更深入的技巧的必备条件
    • 动手实践很重要，要增加自己的体力，就只有运动;要增加自己的知识，就是有读书;当然，要增加自己对Linux的认识，大概就只有实践了。
  3. 本章主要是一些了解性的内容。

第二章 主机规划与磁盘分区

  1. 开机流程到操作系统之前的动作
    1. BIOS：开机主动执行的固件，会认识第一个可开机的设备（联系UEFI）
    2. MBR：第一个可开机设备的第一个扇区内的主要开机记录区块，内含开机管理程序
    3. 开机管理程序（boot loader）：一只可读取核心文件来执行的软件
      • boot loader的主要任务：
        • 提供菜单：使用者可以选择不同的开机项目，这也是多重开机的重要功能
        • 载入核心文件：直接指向可开机的程序区段来开始操作系统
        • 转交其他loader：将开机管理功能交给其他loader负责（这里和安装的双系统有关）
    4. 核心文件：开始操作系统的功能

第四章 首次登录与线上求助page

  1. 开始下达指令

    $ command(指令) -options parameter1(参数1) parameter2 …(参数2)

  2. 基础指令的操作
    1. 显示日期的指令: date
    2. 显示日历的指令： cal
    3. 几个重要的热键 [Tab] ,[ctr]-c,[ctr]-d
      1. [tab] (1)命令补全：连续按两次tab (2)文件补全
      2. [ctr]-c ：指令串终止
      3. [ctr]-d ：键盘输入结束
      4. [shift]+[pageup]|[pagedown]:用来翻页
  3. 错误信息的查看
  4. Linux的线上求助man page与info page
  5. 正确的关机方法
    • 将数据同步写入硬盘中的指令：sync
    • 惯用的关机指令：shutdown
    • 重新开机,关机：reboot,halt,poweroff
    1. shutdown:
      1. -h 将系统的服务停掉后，立马关机
      2. -r 将系统的服务停掉后，重新开机

第二部分 Linux文件、目录与磁盘格式

第五章 Linux文件权限与目录配置

  1. Linux一般将文件可存取的身份分为三个类别，分别是：owner/group/others,且三种身份各有read/write/execute等权限
  2. 文件拥有者，群组，其他人的概念
  3. Linux文件权限的概念
    1. Linux文件属性

      -rw-r–r–. 1 root root 1864 May 4 18:01 initial-setup-ks.cfg
      （档案类型权限）（连接数）（档案拥有者）（档案所属群组）（档案最后被修改的时间）（档案名）

      1. -rwxrwx—
        (档案拥有者权限)(档案所属群组权限)(其他人权限)

      • d 代表目录
      • [-]表示文件
      • [i]表示链接文件
      • [b]设备文件里面的可供存储的周边设备
      • [c]设备文件里的序列设备，如键盘，鼠标
        1. Linux文件权限的重要性
        2. 如何改变文件属性和权限
        3. 常用指令

          chgrp:改变文件所属群组
          chown:改变文件拥有者
          chmod:改变文件的权限，SUID，SGID，SBIT等特性

        4. 改变所属群组，chgrp

          chgrp [-R] dirname/filename …选项与参数
          chgrp users initial-setup-ks.cfg #改变所属群组为users

        5. 改变文件拥有者，chown

          chown [-R] 帐号名称 文件或目录
          chown [-R] 帐号名称：群组名称 文件或目录
          chown bin initial-setup-ks.cfg
          ls -l
          -rw-r–r–. 1 bin users 1864 initial-setup-ks
          将initial-setup-ks.cfg的拥有者改回为root
          chown root:root initial-setup-ks.cfg

          1. 什么时候需要改变文件的群组与拥有者？
            • 复制文件给你之外的其他人时

              cp 来源文件 目标文件(cp 会复制执行者的属性和权限，复制给其他用户就需要修改权限才能使用)

        6. 改变权限 chmod
          1. 有两种方法，分别可以采用使用数字或者是符号来进行权限的变更
          2. 数字类型改变文件权限
            1. r:4
            2. w:2
            3. x:1

              chmod [-R] xyz 文件或目录
              chmod 777 .bashrc
              ls -l
              -rwxrwxrwx.1 root root 2013 .bashrc

            4. 不想让其他人看到： 740
          3. 符号类型改变文件权限
            1. user - u
            2. group -g
            3. others - o
            4. 假如我们要设置一个文件权限为 “-rwxr-xr-x“

              chmod u=rwx,go=rx .bashrc

            5. 假如设置文件权限为 ”-rwxr-xr–“

              chmod u=rwx,g=rx,o=r .bashrc

            6. 将权限去掉，而不变更其他已存在的权限,比如拿掉全部人的执行权限

              chmod a-x .bashrc

        7. 目录与文件的权限意义：注意目录的执行的权利设置
        8. Linux目录配置的依据–FHS
          • /(root,根目录)：与开机系统相关
          • /usr(unix software resource):与软件安装/执行相关
          • /var(variable):与系统运行过程有关
        9. 绝对路径与相对路径

          第六章 Linux文件与目录管理

  4. 目录的相关操作
    1. . 代表此层目录
    2. .. 代表上一层目录
      • 代表前一个工作目录
    4. ～ 代表目前使用者身份所在的主文件夹
    5. ～account 代表account这个使用者的主文件夹
  5. 几个常见的处理目录的指令：
    1. cd :变换目录
    2. pwd: 显示目前的目录
    3. mkdir: 创建一个新的目录
    4. rmdir: 删除一个空的目录
  6. 关于可执行文件路径的变量：$PATH
  7. 文件与目录管理
    1. 文件与目录的检视： ls
  8. 复制，删除与移动
    1. cp (复制文件或目录)

      cp 来源文件 目标文件

    2. rm(移除文件或目录)

      rm [-fir] 文件或目录

    3. mv(移动文件或目录，或更名)
      mv [-fiu] 源文件 目标文件
  9. 文件内容查询
    1. cat 由第一行开始显示文件内容
    2. tac 从最后一行开始显示
    3. nl 显示的时候顺便输出行号
    4. more 一页一页的显示文件内容
    5. less 与more相似，但可以往前翻页
    6. head 只看头几行
    7. tail 只看尾巴几行
    8. od 以二进制的方式读取文件内容
  10. 修改文件时间或创建新文件
    1. 三个时间
      • mtime 文件变更时间
      • ctime 文件状态改变时间，如权限属性更改
      • atime 文件被取用就会更新这个读取时间
    2. touch [-acdmt] 文件
  11. 文件默认权限：umask

    umask -S

  12. 观察文件类型：file
  13. 指令与文件的搜寻
    1. 指令文件名的搜寻
      1. which(寻找可执行文件)
    2. 文件文件名的搜寻
      1. whereis(由一些特定的目录中寻找文件文件名)
      2. locate -l 5 passd(找出系统中所有与passwd有关的文件名，且只列出5个)
      3. updatedb (更新数据库)
      4. find()
